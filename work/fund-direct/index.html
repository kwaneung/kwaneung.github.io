수익증권 직접판매

법인 차세대·개인 모바일 투자 채널을 XPlatform·JSP·Spring Boot로 프론트·백엔드 전 구간 구축하고, AML·대외 API·운영 안정화까지 맡았습니다.

Role
Full-stack / SM
Focus
신규 구축 · 규제 · 운영
Stack
JSP · Spring Boot
Type
Fund Direct Sales

문제

법인 차세대와 개인 모바일을 새로 올려야 했고, AML·대외기관 API, 잔고 원장 배치, DB 세션 안정성, 장애 시 알림·롤백 방지까지 금융 운영 제약이 컸습니다. FE와 BE를 구간 넘어 한 흐름으로 다뤄야 하는 환경이었습니다.

솔루션

  • 법인·개인 채널별 화면·기능을 신규 구축
  • XPlatform / JSP / Spring Boot로 FE·BE를 한 흐름으로 설계·구현
  • AML·대외 API 연동을 방화벽·연결 로직까지 포함해 제품 기능으로 취급
  • Spring Cron으로 잔고 원장 등 주기 처리
  • JDBC → JNDI 세션 전환과 관리자 알림·롤백 방지로 운영 품질 강화

결정의 포인트

  • 규제·연동을 화면 바깥 부가 작업이 아니라 제품 기능으로 설계
  • 세션·알림·롤백 방지로 장애를 운영 가능한 범위로 제한
  • 화면·API·Cron·세션을 같은 시스템 관점에서 묶어 구간 사이 빈틈을 줄임

성과

  • 금융권에서 FE·BE·배치·연동 end-to-end 구축
  • AML·대외 API·원장 배치를 포함한 실무 경험
  • 출시 후 운영 품질까지 책임지는 구조 정착
직판 시스템의 법인·개인 채널을 전 구간 구축하고, 규제·연동·배치·운영 안정화까지 포함해 출시 후에도 버티는 시스템을 만들었습니다.
