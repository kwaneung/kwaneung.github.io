우리카드 유니버셜

우리금융지주 슈퍼앱에 들어갈 우리카드 WebView를 구축하고, 외부 원트래블 Next.js 소스를 내재화해 자체 운영·네이티브 WebView 정합까지 맞췄습니다.

Role
채널계 · WebView / 내재화
Focus
슈퍼앱 · 대출 UX · 소유권
Stack
React · Next.js · TS
Type
Financial Hybrid

문제

프로젝트는 두 축이었습니다. 슈퍼앱 안에 우리카드 영역을 WebView로 넣으면서 금융 UI/UX·규정을 지켜야 했고, 외부에 있던 원트래블 Next.js는 수정 속도와 소유권이 떨어져 내재화가 필요했습니다.

솔루션

  • 우리은행 UI/UX 가이드 기반 WebView 화면·플로우 대응
  • 대출 영역 UI/UX·기능 고도화
  • Promise AJAX로 콜백 지옥을 정리하고 공통 컴포넌트·API 재구성
  • 원트래블 Next.js 소스·웹서버를 사내로 이전 후 자체 운영
  • 네이티브 WebView 규약에 맞게 라우팅·연동·UI 재맞춤

결정의 포인트

  • 성격이 다른 두 축을 WebView 구축과 Next.js 소유권 이전으로 범위를 고정
  • 내재화 후 WebView 정합을 별도 단계로 두어 깨짐을 방지
  • 구조 정리(Promise AJAX·공통 모듈)를 함께 진행해 이후 유지보수 비용을 줄임

성과

  • 슈퍼앱용 우리카드 WebView 구축
  • 외부 Next.js 의존을 제거하고 자체 운영 체계로 전환
  • 공통 모듈화로 재사용성·유지보수성 개선
슈퍼앱 WebView 구축과 원트래블 Next.js 내재화를 병렬로 완수하며, 대출 UX 고도화와 공통 모듈 정리까지 함께 수행했습니다.
